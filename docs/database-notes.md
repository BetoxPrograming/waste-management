# 🗄️ Database Notes

This document describes the database structure and implementation notes used in the **WasteManagement** academic project.

---

# 📌 Database Engine

| Component | Technology |
|---|---|
| Database Engine | MySQL |
| Local Environment | XAMPP |
| Administration Tool | phpMyAdmin |

Database name:

```text
db_gestionresiduos
```

SQL source file:

```text
database/bd_gestionresiduos.sql
```

---

# 📂 Main Tables
> [!NOTE]
> This project was developed as an academic prototype in Costa Rica.
> For that reason, the database name, table names, and field names were originally written in Spanish.
> The documentation is written in English to make the repository easier to review, but the original database structure was preserved to avoid changing the delivered academic implementation.
## usuarios

Stores registered users and community information.

| Field | Description |
|---|---|
| id | User identifier |
| nombre | User name |
| correo | Login email |
| contraseña | User password |
| comunidad | Community association |

> [!Note]
> The current implementation does not include role management.
> Administrative profiles are not implemented in the original academic version.

---

## reportes

Stores environmental reports generated by users.

Examples:

- Improper waste handling
- Collection issues
- Community incidents
- Environmental observations

Typical fields:

| Field | Description |
|---|---|
| id | Report identifier |
| id_usuario | Linked user |
| descripcion | Report description |
| fecha | Report creation date |

---

## recolecciones

Stores waste collection requests.

Examples:

- Special waste pickup
- Recycling requests
- Community collection activities

Detected fields:

| Field | Description |
|---|---|
| id | Request identifier |
| id_usuario | Linked user |
| tipo_residuo | Waste type |
| estado | Current status |
| fecha_solicitada | Requested date |
| fecha_confirmada | Confirmation date |

---

## lugares_mapa

Stores locations displayed in the interactive map.

Examples:

- Recycling centers
- Biodigesters
- Community events

Structure:

| Field | Description |
|---|---|
| id | Location identifier |
| nombre | Location name |
| descripcion | Additional information |
| tipo | Category |
| latitud | Latitude |
| longitud | Longitude |

---

# 🔒 Security Notes

Current implementation uses:

- Local XAMPP environment
- Direct MySQL connection
- Basic authentication flow

Production deployment improvements recommended:

- Password hashing
- Role management
- Validation layer improvements
- Prepared statements review
- Session handling review

## Seed Data Notes

The database exported with this repository already contains sample data.

This was intentionally added to provide a small interactive environment immediately after importing the SQL file.

Current seeded content includes:

| Table | Purpose |
|---|---|
| usuarios | Demo users and interaction profiles |
| reportes | Environmental reports |
| recolecciones | Waste collection requests |
| lugares_mapa | Community locations shown on the map |

### Reserved User IDs

> [!NOTE]
> User ID `1` is reserved as the base administrator/demo profile, but the project does not implement a formal role-based administrator system.
>
> User ID `2` is reserved for anonymous reports. Any report submitted without an active session is linked to this user.
>
> The anonymous user is internally used by:
>
> - Anonymous reports
> - Anonymous collection requests
