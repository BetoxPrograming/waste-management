# 🧩 Features

This document provides a functional overview of the modules implemented in the **WasteManagement** project.

---

# 🏠 Home Module

The home page acts as the entry point of the application.

Main functions:

- Project introduction
- Navigation access
- Quick access buttons
- Login and registration entry points

Purpose:

Provide a centralized interface for community interaction.

---

# 👤 Authentication Module

The authentication system manages user access.

Features:

- User registration
- User login
- Session handling
- Community association

Main files:

```text
usuarios/
```

Purpose:

Allow users to participate in environmental activities and reports.

---

# 📝 Environmental Reports Module

Allows users to report environmental issues.

Examples:

- Waste accumulation
- Collection failures
- Improper disposal
- Community incidents

Features:

- Report creation
- Database storage
- User association

Main files:

```text
reportes/
```

Purpose:

Support environmental reporting and citizen participation.

# 🚛 Collection Requests Module

Allows users to request waste collection activities.

Features:

- Waste type selection
- Request registration
- Request persistence
- Status tracking

Examples:

- Recycling pickup
- Special collection requests
- Community support activities

Main files:

```text
reportes/
```

Purpose:

Improve local waste collection participation.
